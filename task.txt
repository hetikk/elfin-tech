Необходимо спроектировать и реализовать прототип процесса для регистрации пользователя с ручным или автоматическим подтверждением на стороне площадки в зависимости от результатов проверки.

Прототип должен поддерживать следующую функциональность:
— Получение REST запроса с пользовательскими данными как триггер для старта процесса
— Сохранение пользовательских данных (Данные о пользователе могут храниться в любом виде, на усмотрение выполняющего задание - в памяти, в базе H2, как пользователь camunda, etc)
- Проверка почты пользователя на домен в соответствии со справочником, который хранится в Elasticsearch (нужно создать и наполнить индекс тестовыми данными)
— Подтверждение регистрации пользователя проверяющим (пользовательская задача - визуальный интерфейс не обязателен), если проверка почты со справочником не прошла
— Оповещение по почте об успешной регистрации

Используемые технологии: Java, Camunda BPMS, Spring Framework

Результат выполнения предполагает наличие следующих артефактов:
— Принципиальная схема взаимодействия используемых в проектируемом процессе модулей и минимальная объектная модель;
— Разрабатываемая часть в виде отдельного проекта процессного приложения на языке java;
— Инструкции по настройке и локальному запуску;

Все артефакты должны храниться в системе git (возможно использование GitHub, GitLab, etc)
